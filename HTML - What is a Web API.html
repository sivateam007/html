HTML - What is a Web API?

एक वेब API एक डेवलपर की सपना है।

  • यह ब्राउज़र की कार्यक्षमता का विस्तार करता है
  • यह जटिल कार्यों को बहुत सरल बनाता है
  • यह जटिल कोड के लिए सरल सिंटैक्स प्रदान करता है

वेब API क्या है?

API का अर्थ है Application Programming Interface (एप्लिकेशन प्रोग्रामिंग इंटरफेस)। एक API एक प्रकार का इंटरफेस है जो डेवलपर्स को किसी एप्लिकेशन, ऑपरेटिंग सिस्टम या अन्य सेवाओं के विशिष्ट फीचर्स या डेटा तक पहुंचने की अनुमति देता है, जिसमें कार्यों और उप-कार्यों का संग्रह शामिल है।

एक वेब API वेब के लिए एप्लिकेशन प्रोग्रामिंग इंटरफेस है।

API की भूमिका:

API विभिन्न सॉफ्टवेयर घटकों को एक दूसरे के साथ संवाद करने की अनुमति देने वाले "मध्यस्थ" हैं। वे किसी एप्लिकेशन के आंतरिक कार्यों को प्रकट किए बिना, उसके कार्यों का उपयोग करने के लिए डेवलपर्स को अनुमति देते हैं।

HTML API

सभी ब्राउज़रों में जटिल कार्यों का समर्थन करने और डेटा तक पहुंचने में मदद करने के लिए अंतर्निहित वेब API का एक सेट है।

यहां कुछ प्रमुख HTML5 API हैं:

1. Geolocation API

यह API उपयोगकर्ता के वर्तमान स्थान (अक्षांश और देशांतर के साथ) तक पहुंचने के लिए उपयोग किया जाता है।

2. Drag and Drop API

यह API आपको ब्राउज़रों में ड्रैग-एंड-ड्रॉप सुविधाओं का उपयोग करने में सक्षम बनाता है।

3. Web Storage API

इस API में ब्राउज़रों को कुंजी/मान जोड़े संग्रहीत करने की अनुमति देने के तरीके हैं (कुकीज़ की तुलना में अधिक सहज तरीके से)।

4. Web Workers API

यह API जावास्क्रिप्ट पृष्ठ के प्रदर्शन को प्रभावित किए बिना पृष्ठभूमि में चलने की अनुमति देता है। जब वेब वर्कर पृष्ठभूमि में चल रहा होता है, तो उपयोगकर्ता कुछ भी कर सकते हैं: क्लिक करना, चीजें चुनना आदि।

5. Server-Sent Events API

यह API एक वेबपेज को सर्वर से स्वचालित रूप से अपडेट प्राप्त करने की अनुमति देता है।

6. Canvas API

यह API आपको जावास्क्रिप्ट के माध्यम से तुरंत ग्राफिक्स बनाने की अनुमति देता है।

HTML API का उपयोग करने के बारे में नोट्स

एक HTML API को लागू करते समय, आप हमेशा:

ब्राउज़र क्षमता जांचें

हमेशा जांचें कि लक्ष्य ब्राउज़र API का समर्थन करते हैं। यदि कोई ब्राउज़र इसका समर्थन नहीं करता है तो हमेशा एक स्क्रिप्ट या संदेश जोड़ें जो चलाया जाना चाहिए।

// ब्राउज़र समर्थन जांचें
if (navigator.geolocation) {
    // API समर्थित है
} else {
    // API समर्थित नहीं है
    alert("आपका ब्राउज़र Geolocation API का समर्थन नहीं करता");
}

मजबूत त्रुटि प्रबंधन जोड़ें

API की अपेक्षा के अनुसार काम न करने वाली स्थितियों पर ध्यान देने के लिए, मजबूत त्रुटि प्रबंधन जोड़ें, जो निर्बाध उपयोगकर्ता अनुभव सुनिश्चित करता है।

उपयोगकर्ता की अनुमति मांगें

Geolocation API जैसे संवेदनशील डेटा तक पहुंचने वाले API का उपयोग करते समय (जो उपयोगकर्ता के भौगोलिक स्थान तक पहुंचता है), आगे बढ़ने से पहले हमेशा उपयोगकर्ता की सहमति लें।

तृतीय-पक्ष API

तृतीय-पक्ष API आपके ब्राउज़र में अंतर्निहित नहीं हैं।

इन API का उपयोग करने के लिए, आपको वेबसाइट से कोड डाउनलोड करना होगा।

YouTube API

आपको एक वेबसाइट पर वीडियो दिखाने की अनुमति देता है।

video.play() video.pause()

Twitter API

आपको एक वेबसाइट पर ट्वीट दिखाने की अनुमति देता है।

tweet.load() tweet.embed()

Facebook API

आपको एक वेबसाइट पर Facebook जानकारी दिखाने की अनुमति देता है।

FB.login() FB.api()

तृतीय-पक्ष API कुंजी:

अधिकांश तृतीय-पक्ष API के लिए आपके एप्लिकेशन को प्रमाणित करने के लिए API कुंजी की आवश्यकता होती है। ये कुंजी आमतौर पर API प्रदाता की वेबसाइट पर मुफ्त में प्राप्त की जा सकती हैं।

API उपयोग का उदाहरण

Geolocation API उदाहरण

<!DOCTYPE html>
<html>
<head>
    <title>Geolocation Example - Jassif Team</title>
</head>
<body>

<h1>Geolocation API Example</h1>

<button onclick="getLocation()">मेरा स्थान प्राप्त करें</button>
<p id="demo"></p>

<script>
function getLocation() {
    // 1. ब्राउज़र समर्थन जांचें
    if (navigator.geolocation) {
        // 2. उपयोगकर्ता की अनुमति मांगें
        navigator.geolocation.getCurrentPosition(showPosition, showError);
    } else {
        // 3. त्रुटि प्रबंधन
        document.getElementById("demo").innerHTML = 
            "आपका ब्राउज़र Geolocation API का समर्थन नहीं करता।";
    }
}

function showPosition(position) {
    // API ने सफलतापूर्वक प्रतिक्रिया दी
    document.getElementById("demo").innerHTML = 
        "अक्षांश: " + position.coords.latitude + 
        "<br>देशांतर: " + position.coords.longitude;
}

function showError(error) {
    // 4. मजबूत त्रुटि प्रबंधन
    switch(error.code) {
        case error.PERMISSION_DENIED:
            document.getElementById("demo").innerHTML = 
                "उपयोगकर्ता ने स्थान अनुमति अस्वीकार कर दी।";
            break;
        case error.POSITION_UNAVAILABLE:
            document.getElementById("demo").innerHTML = 
                "स्थान जानकारी उपलब्ध नहीं है।";
            break;
        case error.TIMEOUT:
            document.getElementById("demo").innerHTML = 
                "स्थान अनुरोध समय समाप्त हो गया।";
            break;
        case error.UNKNOWN_ERROR:
            document.getElementById("demo").innerHTML = 
                "एक अज्ञात त्रुटि हुई।";
            break;
    }
}
</script>

</body>
</html>
